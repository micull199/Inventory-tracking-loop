# Changelog

High-level "what shipped" log. One line per slice. Newest at the top.

## Unreleased

- **S5** — Taxonomy: custom field defs per leaf node (Manager-owned). New `/admin/taxonomy/{node_id}/fields[?show=…]` list and `/new` form, `POST /admin/taxonomy/{node_id}/fields` create, plus flat `/admin/taxonomy/fields/{id}/edit|archive|unarchive` for the rest. Migration `0006_create_taxonomy_field_defs.py` adds the table with seven field types (text, number, decimal, date, boolean, select, multiselect), JSON `options_json`, `required` boolean, and partial-unique-by-name *and* by-key indexes scoped per node (across active + archived rows). Keys auto-derive from names (lowercase / underscored / alnum-only). Field defs only attach to *leaves*: a top-level node with no active sub-categories, or any sub-category. Creating a sub-category under a node with active field defs is now blocked (400) — manager must archive the defs first. Audit vocabulary: `taxonomy_field_def.{created,updated,archived,unarchived}`.
- **S2** — Locations CRUD (Manager-owned): new `/admin/locations` server-rendered list / new / edit / archive / unarchive routes behind `require_role(Role.MANAGER)`. Migration `0004_create_locations.py` adds the `locations` table (`name` unique across active + archived rows). Mirrors the supplier route shape — same audit semantics (`entity_type="location"` + sparse update diffs) and same flash-after-mutation UX. The base layout gains a "Locations" nav link visible to Manager + Admin.
- **S1** — Suppliers CRUD (Manager-owned): new `/admin/suppliers` server-rendered list / new / edit / archive / unarchive routes behind `require_role(Role.MANAGER)`. Migration `0003_create_suppliers.py` adds the `suppliers` table (`name` unique). Every mutation writes an `audit_log` row with `entity_type="supplier"` and a sparse before/after diff for updates. The base layout gains a one-shot flash region (`role="status"`) and a "Suppliers" nav link visible to Manager + Admin.
- **F3** — Audit log infrastructure (append-only): new `audit_log` table (Alembic migration `0002`) + `record_audit()` helper in `app/audit.py` that flushes a row in the caller's transaction. Wired retroactively into `upsert_user_from_userinfo` (logs `user.created` and, for the bootstrap admin promotion, `user.bootstrap_admin_granted`) and the admin role/status mutations (`user.role_assigned`, `user.status_changed`). DB-level immutability triggers reject UPDATE/DELETE on `audit_log` for both SQLite and Postgres.
- **F2.1** — Admin user-management UI: `/admin/users` is now an HTML page where admins can assign roles and change account status (pending → active, active → disabled). New POST routes `POST /admin/users/{id}/role` and `POST /admin/users/{id}/status` with server-side guards (admin can't demote or disable themselves, can't activate a roleless user). Pending users sort to the top of the list.
- **F2** — Google SSO + users + roles: `users` table (Alembic migration), `Role` and `UserStatus` enums, Authlib OAuth login + callback, signed-cookie sessions, `require_role` dependency, role-gated `/admin/users`, anonymous/pending/welcome index page, dev/test-only login backdoor for Playwright. `SECRET_KEY` and Google credentials are now required when `APP_ENV=prod`.
- **F1** — Project skeleton and verification harness: FastAPI app with `/health`, SQLAlchemy + Alembic wired (SQLite dev), pydantic-settings config, pytest unit + integration + Playwright e2e harness, `make check` green end-to-end.
